# Historical Context

## Lineage

Validatorium continues the encyclopedia tradition associated with Diderot and
d'Alembert: systematize factual knowledge, make it public, and make it verifiable.

## The Standard

The domain is observable reality. The method is the scientific method:

- observation;
- evidence; and
- reproducible verification.

Every stored premise must have evidence that supports it. Debate, consensus,
voting, credentials, popularity, and authority do not determine validity.

From that foundation, higher-level systems can construct encyclopedia articles,
textbooks, reference works, and other products whose factual statements trace to
observable evidence.

## Modern Infrastructure

Validatorium combines established tools: structured ontologies, semantic-web
identifiers, UUID-based records, content-addressed distributed storage, and
machine-readable evidence relationships. Versioned releases preserve exact premise,
term, evidence, asset, and artifact versions.

Runtime evaluators can re-check premises just in time against current observations.
The same infrastructure can support higher-level automated hypothesis testing:
authorized software can execute a method through an instrument, collect
observations, and return provenance-preserving evidence for evaluation. Validatorium
is the factual and evidential foundation beneath that orchestration.

This is not a new standard for truth. It is infrastructure for applying the existing
standard systematically and preserving the evidence chain.
